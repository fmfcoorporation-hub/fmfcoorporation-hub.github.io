BAB 1 – FUNGSI MATEMATIKA & ARITMATIKA

Sum Family

SUM

Deskripsi: Menjumlahkan angka atau rentang sel.

SUMIF

Deskripsi: Menjumlahkan angka dengan satu kriteria.

SUMIFS

Deskripsi: Menjumlahkan angka dengan banyak kriteria.

Average Family

AVERAGE

Deskripsi: Menghitung rata-rata sekumpulan angka.

AVERAGEIF

Deskripsi: Menghitung rata-rata dengan satu kriteria.

AVERAGEIFS

Deskripsi: Menghitung rata-rata dengan banyak kriteria.

Count Family

COUNT

COUNTA

COUNTIF

COUNTIFS

Round Family

ROUND

ROUNDUP

ROUNDDOWN

INT

TRUNC

EVEN

ODD

MROUND

CEILING

FLOOR

RANK

PERCENTILE

QUARTILE

ABS

PRODUCT

QUOTIENT

PI

SIGN

FACT

GCD

LCM

Random

RAND

RANDBETWEEN

Deskripsi: Menghasilkan angka acak dalam rentang tertentu.

BAB 2 – FUNGSI TEKS (TEXT FUNCTIONS)

Manipulasi Dasar

LEFT

Deskripsi: Mengambil sejumlah karakter dari awal teks. Cocok untuk ekstraksi kode, inisial, atau potongan data dari string yang panjang.

RIGHT

Deskripsi: Mengambil sejumlah karakter dari akhir teks. Biasanya dipakai untuk mengambil kode akhir, nomor seri, atau ID unik.

MID

Deskripsi: Mengambil bagian teks dari posisi tertentu dengan panjang tertentu. Sangat fleksibel untuk memotong data di tengah string.

LEN

Deskripsi: Menghitung jumlah karakter dalam sebuah teks (termasuk spasi). Berguna untuk validasi panjang input data.

TRIM

Deskripsi: Menghapus spasi berlebih dan hanya menyisakan satu spasi antar kata. Membantu membersihkan data hasil impor.

REPT

Deskripsi: Mengulang teks sesuai jumlah yang ditentukan. Kadang dipakai untuk membuat pola atau formatting visual sederhana.

TEXT

Deskripsi: Mengubah angka menjadi teks dengan format khusus, misalnya menampilkan tanggal atau mata uang dalam format tertentu.

VALUE

Deskripsi: Mengubah teks yang menyerupai angka menjadi angka riil agar bisa digunakan dalam perhitungan.

Perubahan Bentuk Huruf

UPPER

Deskripsi: Mengubah semua huruf menjadi kapital, sering dipakai untuk standarisasi penulisan data.

LOWER

Deskripsi: Mengubah semua huruf menjadi kecil, berguna untuk konsistensi data teks.

PROPER

Deskripsi: Mengubah huruf pertama tiap kata menjadi kapital, sering dipakai untuk format nama atau judul.

Pencarian dan Posisi

SEARCH

Deskripsi: Mencari posisi teks dalam string tanpa memperhatikan huruf besar/kecil. Cocok untuk pencarian umum.

FIND

Deskripsi: Mencari posisi teks dalam string dengan peka huruf besar/kecil. Lebih spesifik dibandingkan SEARCH.

REPLACE

Deskripsi: Mengganti sebagian teks berdasarkan posisi tertentu. Bisa dipakai untuk memperbaiki data dengan format salah.

SUBSTITUTE

Deskripsi: Mengganti teks tertentu dalam string dengan teks baru. Fleksibel untuk mengganti kata spesifik di teks panjang.

EXACT

Deskripsi: Membandingkan dua teks secara tepat (case-sensitive). Berguna untuk validasi input atau perbandingan data.

Penggabungan Teks

CONCATENATE

Deskripsi: Menggabungkan beberapa teks menjadi satu. Fungsi lama namun masih sering dipakai.

CONCAT

Deskripsi: Versi modern dari CONCATENATE, lebih fleksibel dan mendukung array.

TEXTJOIN

Deskripsi: Menggabungkan teks dengan pemisah (delimiter) dan opsi mengabaikan sel kosong. Sangat efisien untuk data tabel.

Informasi Teks

CLEAN

Deskripsi: Menghapus karakter non-printable dari teks. Membantu membersihkan data impor dari sistem lain.

ISTEXT

Deskripsi: Mengecek apakah nilai adalah teks. Dipakai untuk validasi data.

ISNUMBER

Deskripsi: Mengecek apakah nilai berupa angka. Sering dipakai bersama fungsi teks untuk kontrol alur formula.

T

Deskripsi: Mengembalikan teks dari sebuah referensi. Jika nilai bukan teks, hasilnya kosong.

Fungsi Modern (Excel 365+)

TEXTSPLIT

Deskripsi: Memisahkan teks ke dalam beberapa sel berdasarkan delimiter. Memudahkan pemisahan data seperti CSV.

TEXTBEFORE

Deskripsi: Mengambil teks sebelum pemisah tertentu. Sangat berguna untuk parsing data dengan pola tetap.

TEXTAFTER

Deskripsi: Mengambil teks setelah pemisah tertentu. Membantu ekstraksi data di belakang simbol atau tanda.

VALUETOTEXT

Deskripsi: Mengubah nilai menjadi teks yang terbaca. Membantu debugging atau konversi data ke bentuk string.

UNICHAR

Deskripsi: Menghasilkan karakter berdasarkan kode Unicode. Bisa digunakan untuk simbol khusus.

UNICODE

Deskripsi: Menghasilkan kode Unicode dari karakter. Berguna untuk analisis teks berbasis kode karakter.

BAB 3 – FUNGSI LOGIKA

IF Family

IF

Deskripsi: Mengevaluasi kondisi tertentu lalu mengembalikan nilai berbeda untuk hasil TRUE dan FALSE. Sering menjadi dasar hampir semua logika di Excel.

IFS

Deskripsi: Menangani banyak kondisi sekaligus tanpa perlu menulis IF bersarang. Membuat rumus panjang lebih mudah dibaca.

IFERROR

Deskripsi: Berguna untuk menutupi error seperti DIV/0!, mengganti error dengan teks atau angka yang lebih informatif.

IFNA

Deskripsi: Versi lebih spesifik dari IFERROR, hanya menargetkan error N/A. Umumnya digunakan pada fungsi lookup.

Operator Logika

AND

Deskripsi: Mengevaluasi banyak kondisi, hanya mengembalikan TRUE jika semua syarat benar.

OR

Deskripsi: Menghasilkan TRUE jika minimal satu kondisi benar.

NOT

Deskripsi: Membalikkan hasil logika, TRUE jadi FALSE atau sebaliknya.

XOR

Deskripsi: Mengembalikan TRUE hanya jika jumlah kondisi TRUE bernilai ganjil (misalnya tepat satu benar dari dua kondisi).

Kontrol Logika Tambahan

SWITCH

Deskripsi: Membandingkan sebuah ekspresi dengan beberapa nilai kandidat, lalu mengembalikan hasil yang sesuai. Alternatif praktis untuk IF berlapis.

CHOOSE

Deskripsi: Mengambil nilai dari daftar berdasarkan nomor indeks. Cocok untuk memilih kategori atau opsi terbatas.

CHOOSECOLS (Excel 365)

Deskripsi: Memilih kolom tertentu dari sebuah array berdasarkan nomor indeks.

CHOOSEROWS (Excel 365)

Deskripsi: Sama seperti CHOOSECOLS, tetapi untuk memilih baris.

Fungsi Definisi & Kustom

LET

Deskripsi: Memungkinkan kita menyimpan nilai sementara di dalam formula sehingga bisa dipanggil ulang tanpa ditulis ulang. Membuat formula lebih singkat dan efisien.

LAMBDA

Deskripsi: Membuat fungsi buatan sendiri langsung di Excel, tanpa coding VBA. Dapat digunakan kembali seperti fungsi bawaan.

BYROW

Deskripsi: Mengevaluasi ekspresi logika atau fungsi untuk setiap baris dalam array.

BYCOL

Deskripsi: Mengevaluasi ekspresi logika atau fungsi untuk setiap kolom dalam array.

Fungsi Dinamis (Dynamic Array Logic)

FILTER

Deskripsi: Menghasilkan subset data berdasarkan kondisi logika tertentu. Sangat kuat untuk analisis data dinamis.

UNIQUE

Deskripsi: Menghasilkan daftar nilai unik dari sebuah range, bisa dianggap logika seleksi.

SORT

Deskripsi: Mengurutkan data berdasarkan kriteria tertentu, meski bukan logika murni, sering dipakai bersama FILTER.

SORTBY

Deskripsi: Mengurutkan data berdasarkan range/array lain, cocok jika logika urutannya tidak langsung.

Fungsi Informasi Logika

TRUE

Deskripsi: Mengembalikan nilai logika TRUE, bisa dipakai sebagai konstanta dalam formula.

FALSE

Deskripsi: Mengembalikan nilai logika FALSE, sering digunakan sebagai default pada fungsi logika.

ISLOGICAL

Deskripsi: Mengecek apakah sebuah nilai adalah logika (TRUE atau FALSE).

BAB 4 – FUNGSI TANGGAL & WAKTU

Fungsi Dasar

TODAY

Deskripsi: Mengembalikan tanggal hari ini sesuai sistem komputer, tanpa jam.

NOW

Deskripsi: Menghasilkan tanggal dan waktu saat ini, mencakup jam, menit, dan detik.

Ekstrasi Komponen Tanggal

DAY

Deskripsi: Mengambil angka hari dari sebuah tanggal (1–31).

MONTH

Deskripsi: Menghasilkan nomor bulan dari sebuah tanggal (1–12).

YEAR

Deskripsi: Mengambil tahun dalam bentuk angka empat digit.

WEEKDAY

Deskripsi: Mengembalikan nomor hari dalam seminggu (1–7), bergantung pada sistem awal minggu.

WEEKNUM

Deskripsi: Menghasilkan nomor minggu dalam setahun, sering dipakai dalam laporan mingguan.

ISOWEEKNUM

Deskripsi: Menghitung nomor minggu sesuai standar ISO.

EOMONTH

Deskripsi: Menghasilkan tanggal akhir bulan dari sebuah tanggal tertentu, dengan opsi menambah/kurang bulan.

EOY (Excel 365)

Deskripsi: Memberikan tanggal akhir tahun dari sebuah tanggal.

Pembuatan Tanggal

DATE

Deskripsi: Membentuk tanggal dari kombinasi tahun, bulan, dan hari.

DATEVALUE

Deskripsi: Mengubah teks yang berisi tanggal menjadi nilai tanggal Excel.

EDATE

Deskripsi: Menambahkan atau mengurangi sejumlah bulan dari tanggal tertentu.

YEARFRAC

Deskripsi: Menghitung proporsi tahun yang telah berlalu antara dua tanggal.

Ekstrasi Komponen Waktu

HOUR

Deskripsi: Mengambil jam dari sebuah nilai waktu (0–23).

MINUTE

Deskripsi: Menghasilkan menit dari sebuah waktu (0–59).

SECOND

Deskripsi: Menghasilkan detik dari sebuah waktu (0–59).

TIME

Deskripsi: Menggabungkan jam, menit, dan detik menjadi nilai waktu.

TIMEVALUE

Deskripsi: Mengonversi teks yang berisi jam menjadi nilai waktu Excel.

Perhitungan Selisih Waktu

DATEDIF

Deskripsi: Menghitung selisih antara dua tanggal dalam hari, bulan, atau tahun.

DAYS

Deskripsi: Menghasilkan jumlah hari antara dua tanggal.

NETWORKDAYS

Deskripsi: Menghitung jumlah hari kerja (tidak termasuk akhir pekan & libur).

NETWORKDAYS.INTL

Deskripsi: Versi fleksibel, bisa menentukan pola hari kerja sendiri.

WORKDAY

Deskripsi: Mengembalikan tanggal setelah menambahkan sejumlah hari kerja.

WORKDAY.INTL

Deskripsi: Sama dengan WORKDAY, tetapi dengan pola kerja kustom.

Fungsi Minggu dan Kuartal

WEEKNUM

Deskripsi: Memberikan nomor minggu dalam setahun.

ISOWEEKNUM

Deskripsi: Menyediakan nomor minggu sesuai kalender ISO.

QUARTILE.EXC

Deskripsi: Fungsi statistik untuk menentukan nilai kuartal, kadang digunakan pada analisis periode.

QUARTILE.INC

Deskripsi: Versi inklusif dari QUARTILE.EXC.

Fungsi Modern & Dinamis (Excel 365)

SEQUENCE (tanggal)

Deskripsi: Membuat deretan tanggal berurutan secara otomatis.

TEXTAFTER & TEXTBEFORE

Deskripsi: Membantu parsing tanggal berbentuk teks.

Kombinasi EOMONTH + SEQUENCE

Deskripsi: Dipakai untuk membuat kalender otomatis bulanan.

BAB 5 – FUNGSI LOOKUP & REFERENSI

Lookup Klasik

VLOOKUP

Deskripsi: Mencari nilai pada kolom paling kiri tabel lalu mengembalikan nilai dari kolom target pada baris yang sama (vertikal).

HLOOKUP

Deskripsi: Versi horizontal dari VLOOKUP; pencarian dilakukan di baris teratas tabel dan mengembalikan nilai dari baris target.

LOOKUP

Deskripsi: Fungsi lawas untuk mencari nilai dalam satu baris/kolom dan mengembalikan pasangan terdekat jika tidak ditemukan tepat.

Lookup Modern

XLOOKUP

Deskripsi: Penerus VLOOKUP/HLOOKUP; bisa mencari dari mana pun ke mana pun, dukung nilai default saat tidak ditemukan, dan mode pencarian fleksibel.

XMATCH

Deskripsi: Versi modern dari MATCH dengan mode pencarian persis/terdekat dan dukungan wildcard.

FILTER

Deskripsi: Mengembalikan baris yang lolos kriteria; sering dipakai sebagai lookup multi-hasil yang dinamis.

UNIQUE

Deskripsi: Menghasilkan daftar nilai unik; sering digabung dengan FILTER untuk lookup daftar referensi.

SORT

Deskripsi: Mengurutkan data berdasarkan kriteria tertentu, meski bukan logika murni, sering dipakai bersama FILTER.

SORTBY

Deskripsi: Mengurutkan data berdasarkan range/array lain, cocok untuk pengurutan kondisional.

TAKE

Deskripsi: Mengambil n baris/kolom teratas atau terbawah dari suatu array—berguna setelah FILTER/SORT.

DROP

Deskripsi: Membuang n baris/kolom dari awal/akhir array—membantu menyisakan bagian yang dibutuhkan.

CHOOSECOLS

Deskripsi: Memilih kolom tertentu dari array berdasarkan indeks; semacam ‘lookup kolom’.

CHOOSEROWS

Deskripsi: Memilih baris tertentu dari array berdasarkan indeks; praktis untuk ekstraksi baris spesifik.

HSTACK

Deskripsi: Menyusun beberapa array secara mendatar (menggabung kolom) untuk keperluan referensi/lookup.

VSTACK

Deskripsi: Menyusun beberapa array secara vertikal (menggabung baris) sehingga bisa di-FILTER/LOOKUP sekaligus.

WRAPROWS

Deskripsi: Mengalihkan data 1D menjadi beberapa baris dengan lebar tertentu—membentuk tabel referensi cepat.

WRAPCOLS

Deskripsi: Mengalihkan data 1D menjadi beberapa kolom; kebalikan WRAPROWS tergantung orientasi.

TOROW

Deskripsi: Meratakan array menjadi satu baris; memudahkan pencarian berbasis baris tunggal.

TOCOL

Deskripsi: Meratakan array menjadi satu kolom; sangat cocok untuk daftar referensi konsolidasi.

TAKE + DROP + SORTBY (kombinasi)

Deskripsi: Rantai fungsi modern untuk membentuk hasil lookup yang sangat terarah.

Pencarian Posisi & Pengambilan Nilai

MATCH

Deskripsi: Mengembalikan posisi relatif dari sebuah nilai dalam rentang; pondasi banyak rumus lookup klasik.

INDEX

Deskripsi: Mengambil nilai dari posisi baris/kolom tertentu di suatu rentang; pasangan alami untuk MATCH.

INDEX + MATCH (kombinasi)

Deskripsi: Alternatif populer VLOOKUP karena bisa mencari ‘ke kiri’ dan lebih tangguh saat tabel berubah.

XMATCH + INDEX (kombinasi)

Deskripsi: Versi modern kombinasi INDEX/MATCH dengan opsi pencarian lebih fleksibel.

OFFSET

Deskripsi: Menghasilkan referensi yang digeser dari sel awal; memungkinkan range dinamis untuk lookup.

OFFSET + MATCH (kombinasi)

Deskripsi: Membentuk referensi dinamis ke baris/kolom hasil pencarian posisi.

TRANSPOSE

Deskripsi: Menukar orientasi baris sebelum/sesudah lookup.

Referensi dan Alamat Cell

ADDRESS

Deskripsi: Mengembalikan alamat sel (mis. €B€12) dari nomor baris dan kolom—berguna untuk pembuatan referensi tekstual.

INDIRECT

Deskripsi: Mengubah teks alamat/range menjadi referensi hidup; memudahkan rujukan dinamis lintas sheet.

ROW

Deskripsi: Menghasilkan nomor baris dari sebuah referensi—sering dipakai dalam perhitungan posisi.

ROWS

Deskripsi: Mengembalikan jumlah baris dalam sebuah referensi/array—membantu iterasi dinamis.

COLUMN

Deskripsi: Menghasilkan nomor kolom; berguna untuk offset kolom pada lookup bergeser.

COLUMNS

Deskripsi: Mengembalikan jumlah kolom dalam referensi/array; mendukung logika pergeseran kolom.

AREAS

Deskripsi: Menghitung berapa banyak area pada referensi majemuk (range yang tidak saling berdampingan).

CELL

Deskripsi: Memberikan informasi tentang format/lokasi/konten sel.

FORMULATEXT

Deskripsi: Menampilkan formula sebuah sel sebagai teks.

HYPERLINK

Deskripsi: Membuat tautan aktif ke file, web, atau lokasi dalam workbook; navigasi referensi antarsheet jadi instan.

Pivot, Tabel & Data Model (OLAP)

GETPIVOTDATA

Deskripsi: Mengambil nilai dari PivotTable berdasarkan kombinasi field/item tertentu.

CUBEVALUE

Deskripsi: Mengembalikan nilai agregat dari model data/OLAP berdasarkan anggota/ukuran yang dipilih.

CUBEMEMBER

Deskripsi: Mengembalikan anggota (member) dari kubus/OLAP untuk digunakan dalam rumus lain.

CUBESET

Deskripsi: Membuat set anggota yang bisa dipakai untuk agregasi atau iterasi pada model data.

CUBERANKEDMEMBER

Deskripsi: Mengembalikan anggota ke-n dari sebuah set—berguna untuk ranking dinamis.

CUBESETCOUNT

Deskripsi: Mengembalikan jumlah anggota dalam CUBESET—membantu kontrol literasi laporan.

Utilitas Lookup & Referensi

CHOOSE

Deskripsi: Memilih salah satu nilai dari daftar argumen berdasarkan indeks lookup sederhana berbasis nomor.

RTD

Deskripsi: Mengambil data real-time dari server otomatisasi COM; referensi bernilai waktu nyata untuk dashboard.

LOOKUP + MATCH/INDEX (kombinasi)

Deskripsi: Pola klasik sebelum XLOOKUP—tetap relevan untuk kompatibilitas versi lama.

ISREF

Deskripsi: Menguji apakah sebuah nilai adalah referensi yang valid mendeteksi rujukan sebelum dipakai pada lookup.

HSTACK/VSTACK + XLOOKUP (kombinasi)

Deskripsi: Menggabung tabel lalu melakukan lookup silang secara dinamis.

INDIRECT + ADDRESS (kombinasi)

Deskripsi: Membangun referensi dari teks alamat untuk akses data lintas sheet/workbook.

BAB 6 – FUNGSI KEUANGAN

Nilai Waktu Uang

PV

Deskripsi: Menghitung nilai sekarang (Present Value) dari serangkaian pembayaran di masa depan berdasarkan tingkat bunga tertentu.

FV

Deskripsi: Menghitung nilai masa depan (Future Value) dari investasi saat ini ditambah serangkaian pembayaran dengan bunga tertentu.

NPV

Deskripsi: Menghitung nilai bersih saat ini dari arus kas masa depan menggunakan tingkat diskonto tertentu—dipakai untuk analisis investasi.

XNPV

Deskripsi: Versi presisi dari NPV yang memperhitungkan tanggal aktual tiap arus kas, bukan sekadar periode reguler.

RATE

Deskripsi: Menghitung tingkat bunga per periode dari sebuah investasi atau pinjaman berdasarkan pembayaran dan durasi.

IRR

Deskripsi: Internal Rate of Return; menghitung tingkat pengembalian rata-rata dari serangkaian arus kas reguler.

XIRR

Deskripsi: Versi lebih fleksibel dari IRR yang memperhitungkan jadwal arus kas tidak teratur.

MIRR

Deskripsi: Modified IRR; memperbaiki kelemahan IRR dengan memperhitungkan biaya modal dan tingkat reinvestasi.

Pembayaran & Angsuran

PMT

Deskripsi: Menghitung jumlah pembayaran per periode untuk pinjaman/investasi dengan bunga tetap.

IPMT
